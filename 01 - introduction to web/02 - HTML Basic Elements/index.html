  • block level elements
    elements taking all width and displayed in the page as block level this's means this element taking full width with ability to cange all of this with css
    block elements like
    [div, p, section, article, header,aside, main, footer]
  • inline level elements
    elements taking width equall to with of it's content and displayed in the page as inline level this's means this element can adding another element in the same line inside them with ability to cange all of this with css
    inlinr elements like
    [span, img, a, button, input, code, label, select, time, abbr]
Images and paths
adding images in html by <img src="path" alt="alternative text">
path: A file path describes the location of a file in a web site's folder structure
we have two kinds of files path :
  • relative path :
    A relative file path points to a file relative to the current page like
    < img src="/images/picture.jpg" alt="Mountain" >
  • absolute path :
    An absolute file path is the full URL to a file like
    < img src="https://www.w3schools.com/images/picture.jpg" alt="Mountain" >
- we used the relative path in the web and this's the common way to use path.
